# Typography

The typography system provides a consistent type scale and semantic sizing for all text elements across the portfolio.

## Font Family

The design system uses **Geist** as the primary typeface, with fallbacks to system fonts:

```css
font-family: 'Geist', ui-sans-serif, system-ui, sans-serif
```

## Semantic Typography

Semantic variables provide context-specific sizing:

```css
--font-hero-title: var(--text-4xl)
--font-hero-subtitle: var(--font-text-lg)
--font-section-title: var(--text-3xl)
--font-text-lg: var(--text-lg)
--font-text-md: var(--text-base)
--font-text-sm: var(--text-sm)
--font-caption: var(--text-xs)
```

## Usage

### With Components

Use the `Heading` and `Text` components for semantic typography:

```tsx
import { Heading, Text } from '@/components/base';

<Heading level={1}>Page Title</Heading>
<Heading level={2}>Section Title</Heading>
<Text size="lg">Large body text</Text>
<Text size="md">Regular body text</Text>
<Text size="sm">Small text and captions</Text>
```

### In CSS Modules

```css
.title {
  font-size: var(--text-4xl);
  font-weight: var(--font-weight-bold);
  line-height: 1.2;
}

.body {
  font-size: var(--text-base);
  font-weight: var(--font-weight-normal);
  line-height: 1.5;
}
```

## Line Height

Default line heights:
- **Headings**: 1.2 (tighter for larger text)
- **Body text**: 1.5 (comfortable reading)
- **Small text**: 1.4

## Responsive Typography

Font sizes are mobile-first and use clamp() for fluid scaling where appropriate:

```css
--font-nav-overlay: clamp(2.5rem, 10vw, 4rem)
```

This ensures text scales smoothly between mobile and desktop viewports.

## Accessibility

- All text meets WCAG 2.1 AA standards for readability
- Font sizes use rem units for user scaling preferences
- Sufficient contrast ratios on all backgrounds
- Clear hierarchy with semantic HTML elements
