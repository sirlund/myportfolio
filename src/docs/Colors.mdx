# Colors

The design system uses CSS custom properties for colors, enabling easy theme switching and consistent color usage across all components.

## Color System

All colors are defined using the OKLCH color space for better perceptual uniformity and dark mode support.

## Usage

### In CSS Modules

```css
.button {
  background-color: var(--primary);
  color: var(--primary-foreground);
}

.card {
  background-color: var(--card);
  border: 1px solid var(--border);
}
```

### In Inline Styles

```tsx
<div style={{ background: 'var(--muted)', color: 'var(--muted-foreground)' }}>
  Muted content
</div>
```

## Dark Mode

Colors automatically adapt to dark mode using the `.dark` class. The theme switcher in the toolbar allows you to preview components in both modes.

### Light Mode
- Background: White/light grays
- Foreground: Dark gray/black
- High contrast for readability

### Dark Mode
- Background: Dark gray/black
- Foreground: White/light grays
- Reduced eye strain in low-light environments

## Accessibility

All color combinations meet WCAG 2.1 AA standards for contrast ratios:
- Normal text: 4.5:1 minimum
- Large text: 3:1 minimum
- UI components: 3:1 minimum

Use the a11y addon to verify contrast in your components.
